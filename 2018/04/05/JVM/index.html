类加载器：
 Java虚拟机的类装载就是将包含在类文件中的字节码文件装载到Java虚拟机中，而类装载器却并非是将该字节码文件直接加载到Java虚拟机中，它是将字节码文件(.class文件)的内容转换为Java虚拟机中的类字节码。而实现装载，则需要三个步骤来完成：装载、链接和初始化。
方法区：
 方法区作为Java虚拟机中的共享区域，该区域被所有线程共享，包括函数，接口以及所有的字段都在此区域中定义。
本地方法栈：
 并非所有的 JVM 实现都支持本地（native）方法，那些提供支持的 JVM 一般都会为每个线程创建本地方法栈。如果 JVM 用 C-linkage 模型实现 JNI（Java Native Invocation），那么本地栈就是一个 C 的栈。在这种情况下，本地方法栈的参数顺序、返回值和典型的 C 程序相同。本地方法一般来说可以（依赖 JVM 的实现）反过来调用 JVM 中的 Java 方法。这种 native 方法调用 Java 会发生在栈（一般是 Java 栈）上；线程将离开本地方法栈，并在 Java 栈上开辟一个新的栈帧。
堆：
 堆作为Java虚拟机中最大的区域，应用和对象的数据都存在于这个区域，也是GC的主要回收区域，一个JVM实例只存在一个堆内存。而堆中存储的数据及对象都是由栈帧进行引用，因为栈帧被设计建成后便不能调整其大小，因此也不能存储数组和对象。同时JVM的调优主要针对堆和方法区。
栈：
 每个线程拥有自己的栈，栈包含每个方法执行的栈帧。栈是一个后进先出（LIFO）的数据结构，因此当前执行的方法在栈的顶部。每次方法调用时，一个新的栈帧创建并压栈到栈顶。当方法正常返回或抛出未捕获的异常时，栈帧就会出栈。除了栈帧的压栈和出栈，栈不能被直接操作。所以可以在堆上分配栈帧，并且不需要连续内存。栈的生命周期和线程是同步的，当线程结束时，该栈也会随之摧毁。
程序计数器：
 当前线程执行字节码的信号指示器，线程是私有的，它的生命周期和线程相同分支、循环、跳转、异常处理、线程恢复等基础功能都需要依赖这个计数器来完成。
